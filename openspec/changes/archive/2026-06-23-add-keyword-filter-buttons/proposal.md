## Why

爬蟲爬取的標案數量龐大，使用者（接案公司）需要快速找出與自身業務相關的標案。目前僅能透過眼睛逐筆瀏覽，缺乏依關鍵字快速篩選的機制，導致效率低落。

## What Changes

- 在前端表格上方新增四個關鍵字篩選按鈕：**設備**、**機房**、**醫院**、**儀器**
- 點擊按鈕時，表格僅顯示標案名稱中包含該關鍵字的資料列
- 支援多選（可同時啟用多個關鍵字）
- 新增「全部顯示」按鈕（或再次點擊已啟用的按鈕）可取消篩選
- 篩選邏輯純前端實作，不需重新呼叫 API

## Capabilities

### New Capabilities

- `bid-keyword-filter`：在標案列表頁面提供關鍵字篩選按鈕，讓使用者依標案名稱中的關鍵字（設備、機房、醫院、儀器）過濾顯示結果

### Modified Capabilities

- `bid-table-ui`：標案表格 UI 需新增篩選按鈕區塊，並依篩選狀態動態控制顯示列數

## Impact

- **前端**：React 元件（`App.jsx` 或對應的表格元件）需新增按鈕 UI 與篩選狀態邏輯
- **後端**：無需修改，篩選純在前端進行
- **依賴套件**：無新增依賴
